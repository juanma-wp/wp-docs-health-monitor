number (same as integer)

case 'integer':
		case 'number':
			return typeof value === 'number';
💡

Clarify that integer and number are treated identically by the runtime (both validated with typeof value === 'number'), meaning floats pass the integer check and neither type enforces a whole-number constraint.

  • Change \number\ (same as \integer\) to \number\ — validated identically to \integer\; both accept any JavaScript \number\ value (including floats), unlike JSON Schema where \integer\ is restricted to whole numbers
  • Add a note that the JSON Schema distinction between \integer\ (whole numbers only) and \number\ (any numeric) is not enforced at the block attribute parsing level
