Positives

  • The documentation correctly describes providesContext as an object mapping context names to block attribute names, which matches the getBlockSettingsFromMetadata function in registration.ts where providesContext is listed in allowedFields and WP_Block::refresh_parsed_block_dependents() in PHP iterates $this->block_type->provides_context mapping $context_name => $attribute_name.
  • The documentation correctly states that usesContext should be assigned as an array of context names, which is consistent with WP_Block::refresh_context_dependents() iterating $this->block_type->uses_context as an array via foreach.
  • The documentation accurately notes that 'Block Context is not available to save', which is consistent with the JavaScript architecture where save is a pure serialization function with no access to the block editor store or React context.
